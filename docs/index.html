System Architecture

How Aiko Thinks

A modular neural hub architecture where each satellite handles a specific capability — emotion, memory, voice, vision, and agency.

Neural Hub (Port 8000)
The central brain. Routes all requests, manages state, coordinates satellites, and holds the personality core. Powered by configurable LLM providers.
FastAPI Python WebSocket
Emotion Engine
Analyzes conversational context and updates Aiko's emotional state across 22+ dimensions. Emotions influence her tone, sprite expression, and responses.
Sentiment Analysis State Machine
MemPalace
Persistent memory system that stores conversation history, emotional milestones, user preferences, and relationship progression.
YAML Store Context Window RAG
Voice Synthesis
Pocket-TTS generates emotion-aware speech. Intonation, speed, and warmth adjust based on current emotional state.
Pocket-TTS Emotion-Aware
Vision Module
Gemma-4 powered multimodal understanding. Processes images from chat, screenshots, and camera input for contextual awareness.
Gemma-4 Multimodal
Desktop Shell (Tauri)
Native desktop application with mascot overlay, chat interface, and system tray. Built with Tauri + Kotlin multiplatform for cross-OS support.
Tauri Kotlin Compose

Get Started

Three steps to bring Aiko to life on your machine.

01

Clone the Repository

Grab the latest source from GitHub. Everything you need is in one repo.

git clone https://github.com/omax404/Project-Aiko.git
02

Configure Your Environment

Copy the example env, set your LLM provider keys (Gemini, OpenAI, local), and customize Aiko's personality in the directives.

cp .env.example .env && edit .env
03

Launch Aiko

Run the launcher script. It boots the Neural Hub, all satellites, and the desktop overlay automatically.

python start_aiko_tauri.py
